Chapter 1 - Circuit

Imagine a water pipe with water flowing through it. If you step on the pipe, the flow stops.

[insert picture]

An electrical circuit works the same way: replace the pipe with a wire and the water with electrons. Your foot acts like a switch, turning the flow on and off.

Electricity travels at nearly the speed of light, about 300,000 kilometers per second, which means it could travel around the Earth about 7.5 times in just one second.

So you can think of it as having two states: On and Off.

On means the circuit is closed and electricity is flowing, like water flowing through a pipe.

Off means the circuit is open and nothing is flowing, like a pipe with no water moving through it.

Chapter 2 - Binary Number System

Stop thinking of numbers as values for a moment. Think of them as symbols.

Imagine you discover an ancient Egyptian cave filled with strange symbols. Those symbols represent a number system, even if they don't look like the numbers we use today.

[insert picture]

In a binary number system, there are only two symbols: 0 and 1.

In everyday life, we use the decimal system, which has ten symbols in total: 0 through 9.

You could also have a base-3 number system, which uses the symbols 0, 1, and 2.

A number system is defined by its base. When we say "base 3," we mean a system whose valid symbols range from 0 to 2.

The base-2 number system is called binary.

One important pattern is that the highest symbol in any number system is always one less than the base:

  • Base 2: symbols 0–1, last symbol is 1
  • Base 5: symbols 0–4, last symbol is 4
  • Base 10: symbols 0–9, last symbol is 9

The base tells you how many unique symbols the system uses, not what those symbols look like.

Chapter 3 - Bits and Bytes

A bit is simply a single symbol position in the binary number system.

For example, in base 10, the number 12345 has 5 symbols.

In binary, if you have a number like 0100, it has 4 symbols.

Each one of those symbols is a bit.

If you have a group of 8 symbols (bits), it is called 1 byte.

So 2 bytes = 16 bits.

3 bytes = 24 bits.

Chapter 4 - Binary Addition

In the decimal number system, when you add 1 and 9, you get 10, which uses two symbols.

This happens because there is no symbol higher than 9. When we reach the maximum symbol, it resets to 0, and we carry over a 1 to the next symbol place.

In a binary number system:

When you add 0 and 0, you get 0.

0 and 1 gives you 1.

1 and 1, will give you 10.

Adding "10" and "1" will give you "11".

When happens when you do "11" + "1", you get "100".

Chapter 5 - Binary To Decimal, and back

Do table side by side, let us keep adding 1, in both the decimal and binary number systems.

0 0 0 1 1 1 2 10 2 3 11 3 4 100 4 5 101 5 6 110 6 7 111 7 8 1000 8

Notice how, 8, 4, 2, and 1 have really clean looking pattern with all those zeros.

Following that pattern, we know:

10000 = 16 100000 = 32 1000000 = 64 10000000 = 128 100000000 = 256 1000000000 = 512

To convert a decimal number to binary, try to break it down to that pattern.

So 17 can be 16+1

15 can be 8+4+2+1

So you can represent decimal number as power of 2. And then convert that to binary.

So 15 to binary = 1000 + 100 + 10 + 1 = 1111

So 673 = 512 + 128 + 32 + 1 = 1010100001

You can also convert a binary number to decimal with this process:

101011 = 100000 + 1000 + 10 + 1 = 32 + 8 + 2 + 1 = 43

Sometimes you could have a really big binary number 100000000000, how do you even calculate that. You probably wouldn't want to write the whole table for it. It's too much work and we are lazy. There is a nifty pattern for it:

2^0 = 1 2^1 = 2 2^2 = 4 2^3 = 8 2^(how many zeros in binary suffix) = decimal number

100000000000 = 2^11 = 2048
